# DeceptionGrid Public Demo Checklist

DeceptionGrid is ready to present as a defensive-only KRYNEX Labs module after the following checks pass.

## Demo Story

1. Open the dashboard and show total assets, active assets, 24h events and open alerts.
2. Open Deception Assets and drill into one asset detail page.
3. Generate one safe simulator event from Settings.
4. Show the new event in Events and the generated alert in Alerts.
5. Show Honeytokens and explain that raw token values are shown once and stored only as hashes/prefixes.

## Public Safety Rules

- Keep `DEMO_MODE=true`.
- Use only seeded synthetic IPs and internal-looking demo hosts.
- Do not connect the demo to real customer infrastructure.
- Do not paste real tokens, credentials or logs into screenshots.
- Keep the positioning as a portfolio/open-source defensive SaaS MVP.

## V1.2 Roadmap Readiness

- Keep public demo controls read-only by default.
- Scope ingestion keys to synthetic event workflows.
- Prepare LogForge and Nexus export notes before enabling integration demos.
